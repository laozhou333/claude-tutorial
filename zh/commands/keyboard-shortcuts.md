# 快捷键速查

Claude Code 提供了丰富的键盘快捷键，帮助你高效地与 AI 交互。本文涵盖所有快捷键、文本编辑技巧和 Vim 模式操作。

:::tip 自定义快捷键
Claude Code 支持自定义快捷键绑定。使用 `/keybindings` 命令或编辑 `~/.claude/keybindings.json` 文件进行配置。
:::

## 通用快捷键

最常用的全局快捷键。

| 快捷键 | 说明 | 备注 |
|--------|------|------|
| `Ctrl+C` | 中断当前操作 | 如果 Claude 正在执行工具调用，会取消当前操作 |
| `Ctrl+D` | 退出 Claude Code | 等同于 `/exit` 命令 |
| `Ctrl+L` | 清屏 | 清除终端显示，但不清除会话上下文 |
| `Ctrl+O` | 打开文件选择器 | 选择文件添加到对话上下文中 |
| `Ctrl+G` | 搜索 Git 历史 | 在 Git 日志中搜索并引用特定提交 |
| `Ctrl+R` | 搜索命令历史 | 在过往输入中搜索 |
| `Ctrl+V` | 粘贴 | 粘贴剪贴板内容 |
| `Ctrl+B` | 切换侧边栏 | 在支持的终端中切换文件树显示 |
| `Ctrl+T` | 新建标签页 | 在支持的终端中打开新标签页 |
| `Esc Esc` | 连按两次 Escape | 中断当前操作或清除输入 |
| `Shift+Tab` | 切换焦点 | 在不同 UI 区域间切换 |

### 常见场景

**快速中断失控的操作：**

当 Claude 正在执行一个你想取消的长时间操作时：
1. 按 `Ctrl+C` 中断当前工具调用
2. Claude 会停下来等待你的下一条指令
3. 你可以给出新的方向或修正

**添加文件上下文：**

按 `Ctrl+O` 打开文件选择器，支持模糊搜索。选中的文件会被完整读入对话上下文，这比手动粘贴代码更高效。

:::warning 注意区分
`Ctrl+L` 只清除终端显示，会话上下文仍然保留。如果要清除上下文，请使用 `/clear` 命令。
:::

## 文本编辑

输入消息时的文本编辑快捷键，遵循标准的 Emacs 风格绑定。

| 快捷键 | 说明 | 示例场景 |
|--------|------|----------|
| `Ctrl+K` | 删除光标到行尾的内容 | 快速清除输入的后半部分 |
| `Ctrl+U` | 删除光标到行首的内容 | 清除当前行的输入 |
| `Ctrl+Y` | 粘贴（yank）最近删除的内容 | 配合 `Ctrl+K`/`Ctrl+U` 使用 |
| `Ctrl+A` | 光标移到行首 | 快速跳转到输入开头 |
| `Ctrl+E` | 光标移到行尾 | 快速跳转到输入末尾 |
| `Alt+B` | 光标向左移动一个单词 | 按单词导航 |
| `Alt+F` | 光标向右移动一个单词 | 按单词导航 |
| `Alt+D` | 删除光标右侧的一个单词 | 快速删除下一个单词 |
| `Ctrl+W` | 删除光标左侧的一个单词 | 快速删除上一个单词 |

:::tip Emacs 用户
如果你熟悉 Emacs 编辑器，这些快捷键会让你感到非常亲切。它们在大多数终端程序中通用。
:::

### 文本操作工作流

**快速重写输入：**
1. `Ctrl+U` 清除整行
2. 重新输入

**剪切并重新组织内容：**
1. `Ctrl+K` 剪切到行尾
2. 移动光标到目标位置
3. `Ctrl+Y` 粘贴

## 多行输入

Claude Code 默认按 `Enter` 发送消息。以下方式可以输入多行内容。

| 快捷键 | 说明 | 平台 |
|--------|------|------|
| `\` + `Enter` | 反斜杠换行，最通用的方式 | 全平台 |
| `Option+Enter` | 按住 Option 再按回车 | macOS |
| `Shift+Enter` | 按住 Shift 再按回车 | 部分终端支持 |
| `Ctrl+J` | 插入换行符 | 全平台 |

### 多行输入技巧

**输入多行代码片段：**
```
请帮我写一个函数：\
输入：用户名字符串\
输出：格式化后的欢迎消息\
要求：支持中英文
```

:::tip 推荐方式
不同终端对多行输入的支持不同。`\` + `Enter`（反斜杠换行）在所有环境中都能正常工作，是最可靠的选择。
:::

**粘贴多行内容：**

直接粘贴多行文本（如代码块）到 Claude Code 时，系统会自动识别为多行输入，不需要特殊操作。

## 模型切换快捷键

快速切换模型和推理参数，无需使用斜杠命令。

| 快捷键 | 说明 | 等效命令 |
|--------|------|----------|
| `Option+P` | 切换到上一个/下一个模型 | `/model` |
| `Option+T` | 切换推理努力程度 | `/effort` |
| `Option+O` | 打开模型选择器 | `/model` |

:::tip 快速模型切换
在对话中临时需要更强的推理能力？按 `Option+T` 可以在 low/medium/high 之间快速切换，无需中断思路。
:::

## 快速前缀

在输入框中使用特殊前缀字符快速触发功能。

| 前缀 | 说明 | 示例 |
|------|------|------|
| `/` | 触发斜杠命令自动补全 | `/compact` |
| `!` | 直接执行 Bash 命令 | `!git status` |
| `@` | 引用文件或符号 | `@src/index.ts` |
| `Space`（长按） | 进入语音输入模式 | 长按空格键开始语音输入 |

### 常见用法

**快速运行 Shell 命令：**
```
!npm test
!git log --oneline -5
!ls -la src/
```

使用 `!` 前缀直接执行命令，结果会自动添加到对话上下文中，Claude 可以基于输出进行分析。

**引用特定文件：**
```
@src/components/Button.tsx 这个组件需要加上 loading 状态
```

`@` 前缀会将指定文件的内容加载到上下文中，比手动复制粘贴更高效。

## Vim 模式

Claude Code 内置 Vim 模式支持，输入 `/vim` 开启。

### 启用 Vim 模式

```
/vim
```

开启后，输入框会切换到 Vim 风格的模态编辑。按 `Esc` 进入 Normal 模式，按 `i` 回到 Insert 模式。

### 模式说明

| 模式 | 进入方式 | 说明 |
|------|----------|------|
| **Normal** | `Esc` | 导航和命令模式 |
| **Insert** | `i`, `a`, `o` 等 | 文本输入模式 |
| **Visual** | `v`, `V` | 文本选择模式 |
| **Command** | `:` | 命令行模式（有限支持） |

### Normal 模式 — 导航

| 按键 | 说明 |
|------|------|
| `h` | 左移一个字符 |
| `j` | 下移一行 |
| `k` | 上移一行 |
| `l` | 右移一个字符 |
| `w` | 跳到下一个单词开头 |
| `b` | 跳到上一个单词开头 |
| `e` | 跳到当前单词末尾 |
| `0` | 跳到行首 |
| `$` | 跳到行尾 |
| `^` | 跳到行首非空字符 |
| `gg` | 跳到第一行 |
| `G` | 跳到最后一行 |

### Normal 模式 — 编辑

| 按键 | 说明 |
|------|------|
| `i` | 在光标前进入 Insert 模式 |
| `a` | 在光标后进入 Insert 模式 |
| `o` | 在下方新建一行并进入 Insert 模式 |
| `O` | 在上方新建一行并进入 Insert 模式 |
| `x` | 删除光标处字符 |
| `dd` | 删除整行 |
| `dw` | 删除一个单词 |
| `d$` | 删除到行尾 |
| `cc` | 修改整行（删除并进入 Insert） |
| `cw` | 修改一个单词 |
| `c$` | 修改到行尾 |
| `yy` | 复制整行 |
| `yw` | 复制一个单词 |
| `p` | 在光标后粘贴 |
| `P` | 在光标前粘贴 |
| `u` | 撤销 |
| `Ctrl+R` | 重做 |
| `.` | 重复上一次操作 |

### Normal 模式 — 搜索

| 按键 | 说明 |
|------|------|
| `/pattern` | 向前搜索 |
| `?pattern` | 向后搜索 |
| `n` | 跳到下一个匹配 |
| `N` | 跳到上一个匹配 |

### Visual 模式

| 按键 | 说明 |
|------|------|
| `v` | 进入字符选择模式 |
| `V` | 进入行选择模式 |
| `d` | 删除选中内容 |
| `y` | 复制选中内容 |
| `c` | 修改选中内容 |
| `>` | 增加缩进 |
| `<` | 减少缩进 |

### 数字前缀

在 Vim 命令前加数字可以重复执行：

| 组合 | 说明 |
|------|------|
| `3w` | 向前跳 3 个单词 |
| `5dd` | 删除 5 行 |
| `2dw` | 删除 2 个单词 |
| `10j` | 向下移动 10 行 |

:::tip Vim 模式提示
Vim 模式在编辑长提示词时特别有用。你可以用 Vim 的高效编辑方式组织复杂的提示，然后在 Normal 模式下按 `Enter` 发送。

进入 Insert 模式后，所有标准的输入和多行操作都正常工作。
:::

### 发送消息

在 Vim 模式下：
- **Normal 模式**：直接按 `Enter` 发送当前输入
- **Insert 模式**：`Esc` 回到 Normal 模式后按 `Enter` 发送

## 自定义快捷键

编辑 `~/.claude/keybindings.json` 文件可以自定义快捷键绑定：

```json
[
  {
    "key": "ctrl+s",
    "command": "submit",
    "description": "发送消息"
  },
  {
    "key": "ctrl+shift+c",
    "command": "copy_last_response",
    "description": "复制上一条回复"
  }
]
```

使用 `/keybindings` 命令可以在交互界面中查看和修改绑定。

:::warning 终端兼容性
部分快捷键可能被终端模拟器自身占用。例如 `Ctrl+T` 在某些终端中会打开新标签页而不是传递给 Claude Code。如遇冲突，请在终端设置中调整或选择其他快捷键。
:::
